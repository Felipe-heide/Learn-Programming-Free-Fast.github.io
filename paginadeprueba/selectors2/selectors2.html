CSS (Cascading Styles Sheet) it is called like this because the computer reads the code from top to bottom. Now we can make the question what happens if I first select with a class an element put its color to black and later on the code I select it again with an element selector and change the color to red?








I will explain the the specification. The specification is how important is the declaration, a class selector is more important than an element selector and an id selector is more important than both of them.







So if you first put an id selector and put the color of an element to red, then an element selector or a class selector can not change it. The selector with less importancy is the universal selector you must put '*' before the '{}'. So the list goes like this 'id', 'class', 'element' and 'universal', this are the selectors that are the most important.










There is one last thing I want to show you, '!important', you can put this on a declaration like this 'color: red !important;'. This will make the declaration very important but, a declaration with an element selector with '!important' will still be less important than a declaration with a class selector with '!important'.
